# Definition of Done (DoD)

A feature or release is considered done when:

1. The app runs fully in the browser with no server-side PDF processing.
2. PDF upload and rendering are functional via PDF.js.
3. Section segmentation runs through outline, TOC, heading, and per-page fallback.
4. Language range detection and filtering are available.
5. Sections open the correct PDF page with optional highlight overlays.
6. Draft steps are generated when missing and clearly marked.
7. Edit mode supports section renaming, page changes, steps editing, and drawing highlights.
8. Calibration wizard supports TOC offsets and language ranges.
9. Export options are available (iframe, script, ZIP, Markdown).
10. README includes usage, export guidance, and limitations.
